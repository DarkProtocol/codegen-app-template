# i18n (next-intl v4)

## Overview

i18n is **next-intl ^4**. The app runs a **single active locale** chosen at build/boot
time by the `APP_LOCALE` env var. There is **no `[locale]` segment routing, no
middleware, and no locale switcher** — locale is fixed for the process.

- Supported locales: `ru`, `en` (see `messages` in `src/modules/shared/i18n/messages.ts`).
- `defaultLocale = 'en'` (`src/modules/shared/i18n/shared-messages.ts`).
- `defaultTimeZone = 'UTC'` (`src/modules/shared/i18n/messages.ts`).
- `APP_LOCALE` is whitelisted in `next.config.ts` `env:{}` and mirrored in `.env.example`
  (do **not** use `NEXT_PUBLIC_*`).

## Wiring

`next.config.ts` registers the plugin and points it at the request config:

```ts
import createNextIntlPlugin from 'next-intl/plugin'

const withNextIntl = createNextIntlPlugin('./src/modules/shared/i18n/request.ts')

export default withNextIntl(nextConfig)
```

`src/modules/shared/i18n/request.ts` resolves the active locale and returns its bundle:

```ts
import { getRequestConfig } from 'next-intl/server'
import { defaultTimeZone, getAppLocale, messages } from './messages'

export default getRequestConfig(async () => {
    const locale = getAppLocale()

    return {
        locale,
        messages: messages[locale],
        timeZone: defaultTimeZone,
    }
})
```

There is no locale negotiation here — `getAppLocale()` is the single source of truth.

## Message Composition

`src/modules/shared/i18n/messages.ts` merges per-module bundles plus `Shared` into one
object per locale, `as const`, and exports the locale resolver + types:

```ts
import { i18n as authI18n } from '@modules/auth'
import { i18n as dashboardI18n } from '@modules/dashboard'
import { defaultLocale, sharedMessages } from './shared-messages'

export const messages = {
    ru: { ...authI18n.ru, ...dashboardI18n.ru, Shared: sharedMessages.ru },
    en: { ...authI18n.en, ...dashboardI18n.en, Shared: sharedMessages.en },
} as const

export type AppLocale = keyof typeof messages

export { defaultLocale }
export const defaultTimeZone = 'UTC'

export function getAppLocale(): AppLocale {
    const locale = process.env.APP_LOCALE

    return locale && locale in messages ? (locale as AppLocale) : defaultLocale
}
```

`getAppLocale()` validates `process.env.APP_LOCALE` against the bundle and falls back to
`defaultLocale` for an unknown/missing value. Each module bundle spreads its own
top-level namespace (`Auth`, `Dashboard`); `Shared` is nested under the `Shared` key.

`src/modules/shared/i18n/shared-messages.ts` owns the cross-cutting `Shared` namespace
and the `defaultLocale`:

```ts
export const sharedMessages = {
    ru: { Components: { /* ... */ }, Validation: { /* ... */ }, /* ... */ },
    en: { Components: { /* ... */ }, Validation: { /* ... */ }, /* ... */ },
} as const

export type SharedLocale = keyof typeof sharedMessages
export const defaultLocale: SharedLocale = 'en'
```

## Per-Module Bundles

Each feature module owns a flat **root `i18n.ts`** that exports
`export const i18n = { ru, en } as const`, where the top-level key is the module's
namespace. The module barrel (`index.tsx`) re-exports it so `messages.ts` can compose it.

`src/modules/auth/i18n.ts`:

```ts
export const i18n = {
    ru: { Auth: { LoginForm: { submit: 'Войти', invalidCredentials: '...', loginPreparedTitle: 'Логин' } } },
    en: { Auth: { LoginForm: { submit: 'Log in', invalidCredentials: '...', loginPreparedTitle: 'Login' } } },
} as const
```

`src/modules/auth/index.tsx` (and `src/modules/dashboard/index.tsx`) re-export it:

```ts
export { i18n } from './i18n'
```

`src/modules/dashboard/i18n.ts` follows the same shape under the `Dashboard` namespace
(`Layout`, `Menu`, `Table`, `AdminUsers`, `MediaLibrary`, ...).

The **shared module** is the only exception: it uses an `i18n/` folder
(`messages.ts`, `shared-messages.ts`, `request.ts`) instead of a root `i18n.ts`, and is
deep-import-only (`@modules/shared/i18n/*`). The `Shared` namespace groups
`Components`, `Validation`, `Notification`, `ConfirmModal`, `Errors`, `NotFound`.

> New non-shared module checklist: add a root `i18n.ts` with `{ ru, en } as const`,
> re-export `i18n` from its `index.tsx`, and spread it in `messages.ts` for both locales.

## Namespacing

Keys are **hierarchical PascalCase** `Module.Feature.Component` with **camelCase leaf
keys**. Examples drawn from real bundles:

- `Auth.LoginForm.submit`
- `Dashboard.AdminUsers.PasswordModal.titleWithEmail`
- `Dashboard.MediaLibrary.CreateFile.dropzoneHint`
- `Shared.Validation.requiredField`
- `Shared.Components.CopyInput.copy`

Scope translators to the deepest namespace that covers a component so leaf calls stay
short.

## Consumption

Get a scoped translator with `useTranslations('<Namespace>')`. Components routinely hold
**multiple scoped translators** — a feature `t` plus shared ones:

```tsx
// src/modules/auth/features/login-form/index.tsx
const t = useTranslations('Auth.LoginForm')
const sharedT = useTranslations('Shared.Validation')
// ...
sharedT('requiredField')      // validator message
t('submit')                   // button label
```

```tsx
// src/modules/dashboard/features/media-library/components/create-file-modal/index.tsx
const mediaLibraryT = useTranslations('Dashboard.MediaLibrary')
const t = useTranslations('Dashboard.MediaLibrary.CreateFile')
const sharedT = useTranslations('Shared.Validation')
```

**ICU interpolation** — pass values as the second arg:

```tsx
// src/modules/dashboard/features/admin-users/components/change-password/index.tsx
t('titleWithEmail', { email: user.email }) // "Change password: {email}"
```

**ICU plurals** — define in the bundle, pass `count`:

```ts
// dashboard/i18n.ts
folderItems: '{count, plural, one {# item} other {# items}}' // en
folderItems: '{count, plural, one {# объект} few {# объекта} many {# объектов} other {# объекта}}' // ru
```

```tsx
// src/modules/dashboard/features/media-library/components/folder-card/index.tsx
t('folderItems', { count: itemsCount })
```

**Locale-aware dates/numbers** — use `useFormatter()`, never hand-format:

```tsx
// src/modules/dashboard/features/admin-users/components/list/index.tsx
const format = useFormatter()
// ...
format.dateTime(new Date(user.createdAt), { day: '2-digit', month: '2-digit', year: 'numeric' })
```

## Provider Wiring

`NextIntlClientProvider` lives **only** in `src/modules/shared/provider.tsx`, nested
**inside** the Redux `Provider` and **around** `MantineProvider`:

```tsx
<BaseProvider store={store}>
    <NextIntlClientProvider locale={locale} messages={messages} timeZone={defaultTimeZone}>
        <MantineProvider defaultColorScheme={defaultColorScheme} theme={theme}>
            {/* ... */}
        </MantineProvider>
    </NextIntlClientProvider>
</BaseProvider>
```

The **root server layout** (`src/app/layout.tsx`) resolves the locale, sets `<html lang>`,
and passes `locale` + `messages[locale]` down to `SharedProvider`:

```tsx
const appLocale = getAppLocale()

return (
    <html lang={appLocale} {...mantineHtmlProps}>
        {/* ColorSchemeScript in <head> ... */}
        <SharedProvider defaultColorScheme={defaultColorScheme} locale={appLocale} messages={messages[appLocale]}>
            <AuthProvider>{children}</AuthProvider>
        </SharedProvider>
    </html>
)
```

## Type Safety

- Every bundle is declared `as const`, so message shapes are literal types.
- `AppLocale` (`messages.ts`) and `SharedLocale` (`shared-messages.ts`) are exported and
  used to type provider props, e.g. `messages: (typeof messages)[AppLocale]`.
- There is currently **no global next-intl type augmentation** (no `declare global` /
  `IntlMessages`). Leaf-key autocomplete is therefore not enforced at call sites today —
  adding a `global.d.ts` augmenting `next-intl`'s `Messages` against
  `(typeof messages)['en']` would be a valid future improvement, but it is not present.
- To **thread a translator through helpers**, type the param with
  `ReturnType<typeof useTranslations<'Namespace'>>`:

```tsx
// src/modules/dashboard/features/media-library/components/create-file-modal/index.tsx
function getFileValidationError(
    file: File | null,
    maxFileSize: number | undefined,
    t: ReturnType<typeof useTranslations<'Dashboard.MediaLibrary.CreateFile'>>,
): string | null {
    if (!file) return t('fileRequired')
    if (maxFileSize && file.size > maxFileSize) return t('fileTooLarge', { size: formatFileSize(maxFileSize) })
    return null
}
```

## i18n + Forms

Validation messages are **translated strings returned from `@mantine/form` validators**
(`sharedT = useTranslations('Shared.Validation')`), not Zod and not a schema file. A
validator returns `null` when valid and the translated message when invalid:

```tsx
validate: {
    email: (value) => {
        if (value.trim().length === 0) return sharedT('requiredField')
        return isValidEmail(value) ? null : sharedT('invalidEmail')
    },
}
```

The only extracted validation helper is
`src/modules/shared/helpers/validation.ts` (`isValidEmail` / `EMAIL_REGEXP`). API field
errors are bridged into the form by `useApiFormErrors` and surfaced via translated
fallbacks (`Shared.Errors.somethingWentWrong`). See `references/forms-and-errors.md` for
the full form + API-error contract.
